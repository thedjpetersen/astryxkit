# Domain docs

This repository uses a single domain context.

Before architecture, diagnosis, or implementation work, read:

- `CONTEXT.md` at the repository root when it exists.
- Relevant decisions under `docs/adr/` when they exist.

Use the canonical terms from `CONTEXT.md` in modules, interfaces, tests, plans, and documentation. Surface conflicts with an existing ADR explicitly rather than silently overriding it.
